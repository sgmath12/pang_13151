# PLAN (Phase별 목표) - 초안

각 Phase는 실행 가능한(동작하는) 산출물을 목표로 하며, Phase 종료 시점마다 사람이 직접 실행해보고 피드백을 줄 수 있는 상태여야 한다.

---

## Phase 1. 메인 화면

**목표 기능**
- `npm run dev`로 접속 시 메인 화면(`docs/FEATURES/main.md` 기준)이 표시된다.
- 타이틀, 조작법 안내, 시작 트리거(스페이스바 또는 버튼)가 노출된다.
- 시작 트리거 입력 시 게임 화면(빈 화면 또는 임시 화면)으로 전환된다.

**고객님이 확인할 부분**
- 첫 화면 진입 시 타이틀과 안내 문구가 잘 보이는지.
- 시작 트리거를 눌렀을 때 화면 전환이 정상적으로 일어나는지 (아직 실제 게임 플레이는 없음).

---

## Phase 2. 플레이어 이동/발사

**목표 기능**
- 게임 화면에 플레이어 캐릭터가 표시된다.
- 방향키(←/→)로 좌우 이동, 스페이스바로 작살 발사가 동작한다.
- 화면 경계 밖으로 이동/발사체가 나가지 않도록 처리한다.

**고객님이 확인할 부분**
- 방향키 이동이 자연스러운지 (속도, 화면 끝 처리).
- 스페이스바로 작살이 위로 발사되는지, 연타 시 동작이 이상하지 않은지.

---

## Phase 3. 풍선 등장 및 물리 동작

**목표 기능**
- 미션 1 기준(`docs/FEATURES/mission1.md`) 풍선 1~2개가 등장한다.
- 풍선이 중력의 영향을 받아 바닥/벽/천장에 부딪히며 포물선으로 튕긴다.
- 아직 충돌 판정(작살/플레이어)은 없는 단계.

**고객님이 확인할 부분**
- 풍선의 움직임이 자연스럽게 튕기는지 (원작과 유사한 느낌인지).
- 풍선 크기별 속도 차이가 체감되는지 (이 시점엔 큰 풍선만 있을 수 있음).

---

## Phase 4. 충돌 판정 (작살-풍선, 플레이어-풍선)

**목표 기능**
- 작살이 풍선에 맞으면 `docs/FEATURES/game_rule.md`의 분열 규칙(대형→중형→소형→극소형)대로 분열/소멸한다.
- 플레이어가 풍선에 닿으면 목숨이 1개 감소하고 스테이지가 재시작된다.
- 목숨이 0이 되면 게임 오버 화면이 표시된다.

**고객님이 확인할 부분**
- 풍선을 맞췄을 때 크기별로 정확히 분열되는지, 최소 크기에서는 사라지는지.
- 풍선에 닿았을 때 목숨이 줄고 스테이지가 재시작되는지.
- 목숨을 모두 소진했을 때 게임 오버 처리가 되는지.

---

## Phase 5. 미션 1 클리어 흐름

**목표 기능**
- 모든 풍선(분열 포함)을 제거하면 미션 1 클리어로 판정된다.
- 클리어 시 클리어 안내 화면(또는 문구)이 표시된다.
- 메인 화면 → 미션 1 → 클리어/게임오버 화면까지 전체 플레이 루프가 끊김 없이 동작한다.

**고객님이 확인할 부분**
- 모든 풍선을 제거했을 때 클리어 판정이 정확한 타이밍에 뜨는지.
- 메인 화면 진입부터 클리어(또는 게임오버)까지 전체 흐름이 자연스러운지.
- 반복 플레이(재시작) 시에도 안정적으로 동작하는지.

---

## 참고

각 Phase의 상세 규칙은 `docs/PRD.md`, `docs/FEATURES/game_rule.md`, `docs/FEATURES/mission1.md`를 따른다. Phase 순서나 범위는 진행 중 피드백에 따라 조정될 수 있다.
