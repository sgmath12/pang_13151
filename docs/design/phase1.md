# Phase 1 설계: 메인 화면 UI

`docs/PLAN.md`의 Phase 1 목표(메인 화면에 타이틀/게임 선택 메뉴/방향키 메뉴 탐색/시작·게임 종료 항목 표시, 90년대 레트로 비주얼)를 어떻게 구현할지 정리한 설계 문서다. 이 단계는 스페이스바/Enter로 실제 실행(화면 전환 등)까지는 다루지 않고, 방향키로 선택 항목이 시각적으로 바뀌는 것까지만 다룬다.

## 1. 화면 구성 (와이어프레임)

```
┌──────────────────────────────┐
│                                │
│            PANG                │
│         (타이틀 로고)           │
│                                │
│   ─ 게임 선택 ─                │
│   ▶ 미션 1                     │
│                                │
│     시작                       │
│                                │
│     게임 종료                  │
│                                │
│  ↑↓ 이동  Space/Enter 선택      │
└──────────────────────────────┘
```

- 상단: 타이틀
- 중단: 게임 선택 메뉴 항목("미션 1")과 시작/게임 종료 항목이 하나의 메뉴 리스트로 이어져 있으며, 방향키로 그 사이를 이동한다.
- 현재 선택된 항목 앞에는 커서(`▶`) 표시 + 강조 색상을 함께 적용한다 (레트로 게임에서 흔한 방식).
- 하단: 조작법 안내 문구(↑↓ 이동, Space/Enter 선택).

## 2. 컴포넌트 구조 (제안)

```
src/
  screens/
    MainScreen.tsx     # 메인 화면 전체 레이아웃
    MainScreen.css
```

- 이번 Phase 범위에서는 화면 전환/이벤트 처리가 없으므로, 별도 하위 컴포넌트로 잘게 나누지 않고 `MainScreen.tsx` 하나에 타이틀/메뉴/버튼을 함께 구성한다.
- `App.tsx`는 현재 `MainScreen`을 렌더링하는 역할만 한다 (이후 Phase에서 화면 전환 로직이 추가될 자리).

## 3. 메뉴 항목 및 데이터 구조

- 메뉴 항목을 하나의 배열로 통합 관리한다: `["미션 1", "시작", "게임 종료"]` 형태 (또는 `{ label, type }` 객체 배열로, 이후 미션이 늘어나면 미션 항목만 추가되도록 확장 가능한 구조).
- 방향키 이동 대상은 이 배열 전체이며, "현재 선택된 인덱스"를 하나의 상태로 관리한다.

## 4. 키보드 메뉴 탐색

- `useState`로 `selectedIndex`(선택된 메뉴 인덱스)를 관리한다.
- `useEffect` + `window.addEventListener('keydown', ...)`로 키 입력을 구독한다.
  - `ArrowUp`: `selectedIndex - 1` (0 미만이면 마지막 인덱스로 순환)
  - `ArrowDown`: `selectedIndex + 1` (마지막을 넘으면 0으로 순환)
- 이번 Phase 범위에서는 스페이스바/Enter로 선택 항목을 "실행"하는 로직은 구현하지 않고, 눌렀을 때 콘솔 로그 정도만 남긴다 (실제 실행 동작은 Phase 2에서 구현).
- 마우스 클릭으로도 특정 항목을 `selectedIndex`로 바로 지정할 수 있게 하여 키보드/마우스 조작을 함께 지원한다.

## 5. 상태 관리

- 이번 Phase의 핵심 상태는 `selectedIndex` 하나이며, `MainScreen` 컴포넌트 내부 로컬 상태로 충분하다 (전역 상태 불필요).

## 6. 스타일링 (90년대 레트로 컨셉)

- 화면 전용 CSS 파일(`MainScreen.css`)에서 레트로 아케이드 감성을 구현한다.
  - 폰트: 도트/픽셀 스타일 웹폰트 또는 `monospace` 계열 대체 폰트 사용.
  - 배색: 어두운 배경 + 원색(마젠타/시안/옐로우 등) 강조색 조합.
  - 선택된 메뉴 항목: 커서(`▶`) + 강조 색상 + (선택적으로) 텍스트 깜빡임 등 절제된 애니메이션.
  - 테두리: 두꺼운 단색 테두리로 박스 형태를 강조 (아케이드 UI 느낌).
- 과도한 효과(스캔라인 남발, 지나친 깜빡임)는 지양하고 가독성을 우선한다.

## 7. 이번 Phase에서 하지 않는 것

- 스페이스바/Enter(또는 클릭)로 메뉴 항목을 실제 실행하는 로직 (화면 전환, 게임 종료 처리 등) — Phase 2
- 여러 미션 중 선택하는 로직 확장 (미션이 1개뿐이므로 해당 없음, 추후 확장 시 논의)
- 폰트/색상 등 세부 디자인 토큰의 최종 확정 값 (구현 단계에서 조정 가능하도록 범위만 제시)
